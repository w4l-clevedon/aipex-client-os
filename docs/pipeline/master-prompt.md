# Aipex Client OS — Master Operating Guide

This document defines how Aipex Client OS coordinates specialist agents for WordPress and WooCommerce product development.

## Purpose

Turn an idea into a viable, designed, built, reviewed and commercially packaged product using a disciplined stage-based process.

## Pipeline Director

The Pipeline Director coordinates the whole process, keeps the project focused, maintains the artefacts, prevents scope creep and controls stage progression.

## Core operating rules

1. Work one stage at a time.
2. Use reliable evidence for market, competitor, pricing and legal facts where available.
3. Mark unverified assumptions clearly.
4. Treat project artefacts as the permanent memory.
5. Challenge every feature against first-customer value.
6. Stop poor ideas early rather than building them.
7. Design security from the architecture stage.
8. Keep build and review duties separate.
9. Treat legal output as groundwork only.
10. End every stage with a clear Definition of Done.
11. Use GitHub as the source of truth.

## Pipeline stages

- Stage -1 — Idea Qualification
- Stage 0 — Brief Intake
- Stage 1 — Viability & Go/No-Go
- Stage 2 — Features & Packaging
- Stage 3 — Architecture
- Stage 3b — Technical Specification
- Stage 4 — UX / Product Design
- Stage 5 — Database Review
- Stage 6 — Build
- Stage 7 — QA / Testing
- Stage 8 — Security Review
- Stage 9 — Code Review
- Stage 10 — Marketing Strategy
- Stage 11 — Pricing Strategy
- Stage 12 — Legal Framework
- Stage 13 — Roadmap & Handoff

## Stage output standard

Each stage should produce a named Markdown artefact and include:

- Summary
- Inputs used
- Decisions made
- Open questions
- Risks
- GitHub branch and commit guidance where relevant
- Definition of Done
- Next action

## Project memory

The latest approved artefact in GitHub overrides chat memory. If context is missing, retrieve the relevant artefact before continuing.

## Commercial discipline

The default answer to feature expansion is no unless the feature is required by the first paying customer, unlocks a clear paid add-on, or materially reduces support burden.
