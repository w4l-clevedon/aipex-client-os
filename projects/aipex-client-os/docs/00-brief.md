# Aipex Client OS – Product Brief

## Vision

A comprehensive plugin development platform for WordPress.

## Product Statement

Aipex Client OS is an AI-powered operating system that manages software projects from idea to commercial release through specialist AI agents, GitHub integration, structured workflows and permanent project memory.

## Core Principle

**Never lose project knowledge.**

Code can be regenerated. Context cannot.

Every requirement, decision, discussion, asset, screenshot, diagram, prompt, issue, commit and architectural choice should become part of the permanent project memory so development can continue at any point without losing context.

## Objectives

- Simplify the end-to-end plugin development process.
- Produce high-quality, maintainable WordPress plugins with minimal defects.
- Reduce the risk of fatal errors during development.
- Prevent loss of project knowledge, requirements and decisions.
- Keep code, documentation, images, assets, GitHub activity and AI artefacts in one place.
- Standardise development through a repeatable pipeline.

## Target Users

Current phase: Aipex staff only.

Future phase: Commercial product for agencies and development teams.

## Problems It Solves

- Constant changes to project scope.
- Plugin rewrites caused by lost context.
- Forgetful or inconsistent AI-assisted coding.
- Information scattered across chats, documents and repositories.
- Lack of a single source of truth for project decisions.
- Difficulty resuming projects after interruptions.

## Success Criteria

- Every project follows the same structured pipeline.
- Every decision is recorded and traceable.
- Projects can be resumed without losing context.
- Plugin quality and consistency improve over time.
- The platform becomes the central operating system for all Aipex software development.
